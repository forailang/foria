# 10.10 — db (SQLite)

The `db.*` namespace provides SQLite database access. Connections are represented by a `db_conn` handle. All database operations are synchronous from the perspective of the func body — each call awaits completion before returning.

## Handle Type

| Handle | Created by | Used by |
|--------|-----------|---------|
| `db_conn` | `db.open` | `db.exec`, `db.query`, `db.close` |

## Op Table

| Op | Args | Returns | Description |
|----|------|---------|-------------|
| `db.open` | path | db_conn | Open SQLite database; `":memory:"` for in-memory |
| `db.exec` | conn, sql [, params] | dict | Execute SQL (INSERT/UPDATE/DELETE/DDL); returns `{rows_affected}` |
| `db.query` | conn, sql [, params] | list | Execute SELECT; returns list of row dicts |
| `db.close` | conn | bool | Close connection |

### Parameterized Queries

Both `db.exec` and `db.query` accept an optional `params` list for parameterized queries. Parameters are bound positionally to `?` placeholders in the SQL string.

```fa
rows = db.query(conn, "SELECT * FROM users WHERE email = ?", list.append(list.new(), email))
```

## Examples

### Opening and setting up a database

```fa
func SetupDb
  take path text
  emit conn db_conn
body
  conn = db.open(path)
  db.exec(conn, "CREATE TABLE IF NOT EXISTS users (id TEXT PRIMARY KEY, name TEXT, email TEXT)")
  db.exec(conn, "CREATE TABLE IF NOT EXISTS sessions (id TEXT PRIMARY KEY, user_id TEXT, created_at INTEGER)")
  emit conn
done
```

### Inserting a row

```fa
func CreateUser
  take conn db_conn
  take id text
  take name text
  take email text
  emit ok bool
  fail err dict
body
  params = list.new()
  params = list.append(params, id)
  params = list.append(params, name)
  params = list.append(params, email)
  result = db.exec(conn, "INSERT INTO users (id, name, email) VALUES (?, ?, ?)", params)
  affected = obj.get(result, "rows_affected")
  if affected == 0
    fail error.new("INSERT_FAILED", "No rows inserted")
  done
  emit true
done
```

### Querying rows

```fa
func ListUsers
  take conn db_conn
  emit users list
body
  users = db.query(conn, "SELECT id, name, email FROM users ORDER BY name")
  emit users
done
```

### Querying with parameters

```fa
func GetUserByEmail
  take conn db_conn
  take email text
  emit user dict
  fail err dict
body
  params = list.append(list.new(), email)
  rows = db.query(conn, "SELECT id, name, email FROM users WHERE email = ?", params)
  if list.len(rows) == 0
    fail error.new("USER_NOT_FOUND", "No user with email #{email}")
  done
  emit rows[0]
done
```

### Updating a row

```fa
func UpdateUserName
  take conn db_conn
  take user_id text
  take new_name text
  emit ok bool
  fail err dict
body
  params = list.append(list.append(list.new(), new_name), user_id)
  result = db.exec(conn, "UPDATE users SET name = ? WHERE id = ?", params)
  if obj.get(result, "rows_affected") == 0
    fail error.new("NOT_FOUND", "User #{user_id} not found")
  done
  emit true
done
```

### Deleting a row

```fa
func DeleteUser
  take conn db_conn
  take user_id text
  emit ok bool
body
  params = list.append(list.new(), user_id)
  db.exec(conn, "DELETE FROM users WHERE id = ?", params)
  emit true
done
```

### Counting rows

```fa
func CountUsers
  take conn db_conn
  emit count long
body
  rows = db.query(conn, "SELECT COUNT(*) as count FROM users")
  row = rows[0]
  count = to.long(obj.get(row, "count"))
  emit count
done
```

### In-memory database (for tests)

```fa
func RunWithMemDb
  emit result dict
body
  conn = db.open(":memory:")
  db.exec(conn, "CREATE TABLE items (id INTEGER PRIMARY KEY, name TEXT)")
  params = list.append(list.new(), "Widget")
  db.exec(conn, "INSERT INTO items (name) VALUES (?)", params)
  rows = db.query(conn, "SELECT * FROM items")
  db.close(conn)
  emit rows[0]
done
```

### Transactions

```fa
func TransferFunds
  take conn db_conn
  take from_id text
  take to_id text
  take amount real
  emit ok bool
  fail err dict
body
  db.exec(conn, "BEGIN")
  debit_params = list.append(list.append(list.new(), amount), from_id)
  debit = db.exec(conn, "UPDATE accounts SET balance = balance - ? WHERE id = ? AND balance >= ?",
    list.append(debit_params, amount))
  if obj.get(debit, "rows_affected") == 0
    db.exec(conn, "ROLLBACK")
    fail error.new("INSUFFICIENT_FUNDS", "Not enough balance in account #{from_id}")
  done
  credit_params = list.append(list.append(list.new(), amount), to_id)
  db.exec(conn, "UPDATE accounts SET balance = balance + ? WHERE id = ?", credit_params)
  db.exec(conn, "COMMIT")
  emit true
done
```

### Row field access

Each row returned by `db.query` is a dict. Field names match the column names (or aliases) in the SQL:

```fa
rows = db.query(conn, "SELECT id, name, created_at FROM users")
loop rows as row
  id = obj.get(row, "id")
  name = obj.get(row, "name")
  created_at = to.long(obj.get(row, "created_at"))
  log.info("User #{id}: #{name} (created #{to.text(created_at)})")
done
```

## Common Patterns

### Connection per request vs. shared connection

For web servers, open the connection once at startup and pass the `db_conn` handle through your flow. SQLite supports multiple concurrent readers but only one writer at a time.

### Upsert

```fa
db.exec(conn, "INSERT OR REPLACE INTO settings (key, value) VALUES (?, ?)",
  list.append(list.append(list.new(), key), value))
```

### Pagination

```fa
params = list.append(list.append(list.new(), page_size), offset)
rows = db.query(conn, "SELECT * FROM items ORDER BY id LIMIT ? OFFSET ?", params)
```

### Check existence

```fa
params = list.append(list.new(), user_id)
rows = db.query(conn, "SELECT 1 FROM users WHERE id = ? LIMIT 1", params)
exists = list.len(rows) > 0
```

## Gotchas

- `db.open` with a file path creates the file if it does not exist. Use `file.exists` first if you need to distinguish between a new and existing database.
- `db.query` always returns a list, even for a single row. Access with `rows[0]` and always check `list.len(rows) > 0` first.
- Column values returned by `db.query` have types based on SQLite's type affinity. INTEGER columns return `long`, REAL columns return `real`, TEXT columns return `text`. NULL values return `void`.
- Parameterized queries (`?` placeholders) are the only safe way to include user-supplied values in SQL. Never use string interpolation to build SQL — that creates SQL injection vulnerabilities.
- `db.close` should be called when the connection is no longer needed. In-memory databases are destroyed on close.
- SQLite WAL mode is not set by default. For write-heavy concurrent workloads, run `db.exec(conn, "PRAGMA journal_mode=WAL")` after opening.
